Getting Started With IntelliJ IDEA Gosu Plugin

There is a Gosu plugin for the IntelliJ IDEA IDE. The Gosu plugin enables editing, debugging, and running Gosu within the IDE. You can use Gosu code completion, semantic searching, and refactoring tools.
For installation instructions and full feature list for the Gosu plugin for the IntelliJ IDEA IDE, visit:
http://gosu-lang.org
To provide feedback to the Gosu team about this plugin, see “Get Ready for Gosu”.
Refer to the IntelliJ documentation for complete information about:
•
creating new class files
•
organizing modules
•
refactoring tools
•
file management (including Java JAR files)

Install the IntelliJ Gosu Plugin

To install the IntelliJ IDEA Gosu Plugin
1.
Ensure you are using the version of IntelliJ IDEA specified in “System Requirements”.
2.
Ensure you have the version of Java specified in “System Requirements” installed on your operating system.
3.
If you use Mac OS X, you must force IntelliJ to use JDK1.7 instead of JDK1.6 inside the application:
a.
In Finder, click Go menu → Go To Folder and paste in:
   /Applications/IntelliJ IDEA 12.app/Contents
b.
Open the Info.plist file in a text editor.
c.
Look for the string JVMVersion:
 
      <key>JVMVersion</key>
      <string>1.6*</string>
d.
Change it to the values:
 
      <key>JVMVersion</key>
      <string>1.7*</string>
e.
Save the file and quit.
4.
Remove earlier versions of the Gosu plugin and incompatible other plugins such as Groovy:
a.
Start IDEA.
b.
From the IntelliJ IDEA Welcome Dialog, in the Quick Start pane, click Configure → Project Defaults → Project Structure. On Windows, use the keyboard shortcut Ctrl-Alt-Shift-S.
c.
Click SDKs on the left part of the window.
d.
If Default Gosu SDK is visible in the second column, right-click it, then click Delete.
e.
Close the Project Structure dialog.
f.
Open the dialog for Settings, which on Windows is the keyboard shortcut Ctrl-Alt-S.
g.
Click Plugins on the left part of the window.
h.
If Gosu is listed in the plugins list, right-click on it, and then click Uninstall.
i.
If Gosu Debugger is listed in the plugins list, right-click on it, and then click Uninstall.
j.
Ensure that the Groovy plugin is disabled. You do not need to uninstall it, but ensure the checkbox is unchecked. The file extension for Groovy files conflicts with Gosu programs (.gsp) and causes problems.
k.
Close the Settings dialog. Restart IDEA when it prompts you to restart IDEA.
5.
Install the new plugin:
a.
From the IntelliJ IDEA Welcome Dialog, in the Quick Start pane, click Configure → Plugins.
b.
Click Plugins on the left part of the window.
c.
Confirm that Gosu is not listed.
d.
Click Browse Repositories, scroll to the value Gosu. Right-click on Gosu, then click Download and Install. Click Yes in the confirmation dialog.
WARNING Be careful to not choose Gosu Debugger, which is an obsolete incompatible plugin.
If you are installing a development release provided as a ZIP file, instead of clicking Browse Repositories, click Install Plugin from Disk and choose the ZIP file.
e.
Click OK to close the Repository Browser dialog, then click OK to close the Plugins dialog. You will see a progress bar in the lower-right of the main window.
f.
If you wait for the download to complete fully before closing the Plugins dialog, IntelliJ prompts you in a large dialog to restart, in which case restart IDEA immediately.
If you do not wait for the download to complete fully before closing the Plugins dialog, you get a different notification that the download is complete. When the download completes, IntelliJ IDEA displays a green alert in the upper-right part of the window suggesting to restart IntelliJ IDEA. If the alert mentions “plugin error”, do not worry. That message does not necessarily indicate an actual problem. Click the Restart Now link in the alert to restart IDEA.
IMPORTANT After installing the plugin, you must restart IntelliJ IDEA before proceeding. Otherwise, the plugin is not installed correctly.
6.
After IntelliJ IDEA restarts, verify the plugin version number:
a.
Check the Gosu web site for the latest version number.
b.
In IntelliJ IDEA, open the dialog for Settings, which on Windows is the keyboard shortcut Ctrl-Alt-S.
c.
Click Plugins on the left part of the window.
d.
Click Gosu.
e.
Read the description. Verify that the description mentions the expected version number of the Gosu plugin.
7.
Confirm the Default Gosu SDK is installed and it matches the correct JDK:
a.
From the IntelliJ IDEA Welcome Dialog, in the Quick Start pane, click Configure → Project Defaults → Project Structure. On Windows, you can use the keyboard shortcut Ctrl-Alt-Shift-S.
b.
Click SDKs on the left side of the window.
c.
Click Default Gosu SDK. If it is not visible, the Gosu plugin was not successfully installed.
d.
Look in the JDK Used field, confirm that the path is to a local Java directory for a supported version of Java. See “System Requirements”.
e.
Close the Project Structure dialog.
8.
Test basic Gosu plugin functionality with the procedure “Create a New Gosu Project with the IntelliJ IDEA Gosu plugin”.

Create a New Gosu Project with the IntelliJ IDEA Gosu plugin

To create a Gosu project and test basic Gosu plugin functionality
1.
Quit and restart IntelliJ IDEA to ensure that all changes have taken effect.
2.
Close all other projects in IntelliJ, no matter what type they are. With all projects closed, you see only the initial IntelliJ Welcome Dialog.
WARNING In the current early version of the Gosu plugin, it is the defined behavior that only one project can be open at a time. This is a strict requirement to use the Gosu plugin.
You can temporarily disable the Gosu plugin in the IntelliJ IDEA Plugins dialog if you need to view multiple projects. Uncheck the checkbox next to the Gosu plugin in the list. However, ensure you remember to re-enable it again later before attempting more Gosu development. Alternatively, you can create separate instances of IDEA, one for each development project, or one for Gosu development and one for non-Gosu development.
3.
From the IntelliJ IDEA Welcome Dialog, in the Quick Start pane, click Create New Project.
4.
Enter fields in the Create New Project dialog:
a.
In the left column of the dialog, there is a list of module types. Select Gosu Module. The simplest test is a Gosu project with a Gosu program (a .gsp file) as the main file in the project. Creating a Gosu module lets you run a Gosu program.
WARNING Failing to choose Gosu Module is the most common mistake in Gosu plugin setup.
If Gosu Module is not visible, the Gosu plugin is not successfully installed in this instance of IntelliJ IDEA. Review the installation instructions carefully. You may need to uninstall the plugin and reinstall.
b.
In the Project name field, type a project name such as gosutest1.
c.
In the Project location field, choose a location for your IntelliJ project, such as C:\gosu\projects
d.
In the Project SDK field, you must use Default Gosu SDK, which may already be the default
e.
Click the Next button at the bottom of the dialog.
f.
In the Create source directory field, use the default value src then click Next again.
g.
In the picker to select an SDK, you must click Default Gosu SDK again, then click Finish.
5.
In rare cases, the IntelliJ SDK choice does not take effect even if you set it correctly in previous steps. To confirm, go to your project settings immediately and verify the SDK:
a.
Go to the Project Structure window. On Windows, the keyboard shortcut is Ctrl-Alt-Shift-S. On Mac OS X, the keyboard shortcut is Cmd-;.
b.
Under Project, look for the picker Project SDK.
c.
If it is not set to Default Gosu SDK, set it to Default Gosu SDK. If it was not set correctly, after you make this change, you must restart IntelliJ IDEA once more. Be warnred that IntelliJ does not prompt you to restart, but it is still required in this case.
6.
In the project window for your new project, look for the first node in the list with your project name. Confirm that node has the icon of a large letter G, which stands for Gosu. If the project does not have the icon of the letter G, you did not set the project type correctly in the previous steps. It is also possible that the Gosu plugin is not installed correctly.
7.
If necessary, expand the hierarchy on your root project icon so that you see the child node with label src.
8.
Under your project name icon, right-click on the src directory, then click New → Package. For the package name, type test. IntelliJ creates a new top-level package called test.
9.
Right-click on your new top-level package called test, then click New → Gosu Program. For the program name, type HelloWorld. If you do not see the menu item Gosu Program in the New menu, the Gosu plugin is not installed correctly. IntelliJ displays your new program file HelloWorld.gsp, which is initially empty.
10.
In your Gosu program file, type the following:
print("HELLO " + "WORLD")
11.
From the Run menu, click Run.... Select your HelloWorld.gsp from a configuration dialog if it appears.
12.
In the console area, look for text such as:
 
"C:\Program Files\...
 
HELLO WORLD
 
Process finished with exit code 0
13.
If IntelliJ fails with an exception, check the console for an initial line that specifies an IntelliJ JDK, such as:
  "C:\Program Files (x86)\Java\jdk1.6.0_45\bin\java" "-Dtools.jar=C:\Program Files
Confirm that JDK path indicates that IntelliJ is using a supported JDK version, see “System Requirements”.
14.
If you followed these instructions and you still saw errors, post to forums or to file bugs on this Web page:
http://gosu-lang.org/intellij.html

Do Not Depend Directly on Gosu JARs

If you have IntelliJ iml files, confirm that they do not depend on Gosu JARs directly. Access to those APIs is automatic when you correctly configure the SDK to Default Gosu SDK.

The Gosu Scratchpad

In IntelliJ IDEA, within a Gosu project you can at any time use the Gosu Scratchpad window. The Gosu Scratchpad window is essentially a Gosu program in a temporary directory. This temporary Gosu program has access to the type system for all classes and other types in the current project.
In the Tools menu, click Gosu Scratchpad. The keyboard shortcut for this window on Windows is Alt-Shift-S.
You can type any text and then run the program using the commands in the Run menu.

Keep Only One Project Open at a Time

In the current early version of the Gosu plugin for IntelliJ IDEA, it is the defined behavior that only one project can be open at a time. This is a requirement for successful use of the plugin.
You can temporarily disable the Gosu plugin in the IntelliJ IDEA Plugins dialog if you need to view multiple projects. Uncheck the checkbox next to the Gosu plugin in the list. However, ensure you remember to re-enable it again later before attempting more Gosu development.
Alternatively, you can create separate instances of IDEA, one for each development project, or one for Gosu development and one for non-Gosu development.
